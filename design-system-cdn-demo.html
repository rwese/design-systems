Release Notes

  • Version: 1.4
  • Updated: 2026-05-14

This release simplifies documentation pages for short, static reports that need to be readable in a browser and in print.

Contents

  1. Changes
  2. Usage

Changes

The stylesheet now provides a document shell, readable typography, table of contents styling, callouts, code blocks, tables, mobile layout, and basic print support.

Use semantic HTML first. Add ds- classes only for page-level structure and callouts.

Usage

Link the CDN stylesheet and write one focused document per HTML file.

<link rel="stylesheet" href="https://cdn.example.com/design-system-cdn.css">
  1. Use body.ds-page and article.ds-doc.
  2. Keep the summary to one paragraph.
  3. Use normal HTML for links, lists, and code.

Keep generated pages short. Move detailed references to a separate document.
